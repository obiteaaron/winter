1. 参考Akka，以及Actor模型，定义Actor工作节点模型
2. 以RPC框架为核心，注册Actor，构建内网集群
3. 集群之间可负载均衡调用，也可广播调用
4. 定时器的选择：红黑树和最小堆适用于单线程，其核心原因在于如果在多线程环境下，锁红黑树或者最小堆，那就要锁整个，粒度太大了。而时间轮的时间复杂度可以看到点，粒度较小，所以时间轮更适合多线程环境。
   1. 红黑树(单线程)：nginx 
   2. 跳表(单线程)：redis 
   3. 最小堆(单线程)：libevent，go(最小四叉堆)，libev(最小四叉堆)；大部分都是用最小堆来实现定时器。几千几万的任务系统、以及单线程通常选用。
   4. 时间轮(多线程)：netty、kafka、skynet、crontab、powerjob
5. 启动调度的这部分功能和其他的Scheduler是重合的。主要区别是提供主节点执行任务、常驻任务、Map任务、MapReduce任务、DAG任务功能。
6. 可选持久化任务信息和任务实例信息到数据库，方便查看，默认不持久化，也就是无需数据源或者建数据表。
